### 2. 并发不交错 — ✅
真实 HTTP 压测：5 个 `POST /outbox/claim {limit:1}` 并发：
```
{"id":67, cand_id:11, attempts:4}
{"id":68, cand_id:15, attempts:4}
{"id":69, cand_id:14, attempts:4}
{"id":72, cand_id:29, attempts:2}
{"id":73, cand_id:30, attempts:8}
```
每次恰 1 条，5 个 id 全部互异（无双 claim）。backend 硬限 `limit=1` 生效。

扩展端 `content.js` promise-chain mutex（Task 1 + code review C1 修复）+ `sendResponse({queued:true})` 防止 MV3 message port 关闭后再 reap 回滚重派。

**真实 Boss 字符交错** 需人工发真实消息到 Boss IM 目视确认——backend 层保证已锁死。

### 3. 达标自停 — ✅

PUT /settings {target_count:10}                →  complete_count=4 → is_running=true

边界语义正确。
