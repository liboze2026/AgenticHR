# F4 自动采集 · 小白快速上手

## 一次性配置（5 分钟）

### 1. 启动后端
```bash
# 项目根目录
uvicorn app.main:app --host 0.0.0.0 --port 8000
```
保持这个窗口开着。

### 2. 启动前端
```bash
cd frontend && pnpm build && pnpm preview
```
或直接访问 http://localhost:8000/（已挂载前端 dist）。

### 3. 装 Edge / Chrome 扩展
1. 浏览器地址栏输入 `chrome://extensions/`
2. 右上角打开「开发者模式」
3. 点「加载已解压的扩展程序」→ 选 `edge_extension/` 目录
4. 扩展图标出现在工具栏

### 4. 扩展内登录 + 打开自动采集
1. 点扩展图标 → popup 弹出
2. 「服务器地址」保持 `http://127.0.0.1:8000`（默认已填）
3. 输入招聘助手账号密码 → 登录（拿到 JWT，自动存入扩展）
4. 打开「F4 自动采集」开关 → 后台每 **5 分钟** 自动扫一次

## 日常使用（两条路径）

### 路径 A · 手动单聊采集（推荐新手）
1. 去 http://localhost:8000/intake
2. 点某候选人行后的「**开始沟通**」→ 自动打开 Boss 聊天页
3. 扩展自动：
   - 找到该候选人并打开聊天
   - 抓取历史消息
   - 下载 PDF 简历（若对方已发）
   - LLM 抽取 3 个硬槽位：到岗时间 / 面试时段 / 实习时长
   - 若信息全 → 直接归档到简历库
   - 若缺 → 自动发一条合并问题给候选人
4. 右上角「采集」toast 会提示当前状态

### 路径 B · 自动扫描（挂机模式）
1. 开着 Boss 聊天列表页（`/web/chat/index`）
2. 扩展每 5 分钟拉取后端「排队中候选人」列表（按优先级排）
3. 自动在列表里匹配 `data-id` → 逐个点开 → orchestrator 接管

## Dashboard 查看

http://localhost:8000/intake 顶部显示：
- **今日自动采集额度** X / 200（默认 200/天）
- 候选人列表：姓名 / Boss ID / 目标岗位 / 状态 / 进度 / 操作

状态含义：
- `收集中` — 还在抓信息
- `等待回复` — 已发问题，等候选人回
- `待人工` — 问了 3 次还没答齐，人工兜底
- `已完成` — 进简历库
- `已放弃` — PDF 72h 没收到，自动放弃

## 已知行为

- **初装默认「F4 自动采集」关闭**。不打开 → 不会定时扫。首次用户必须手动开。
- **首次触发需用户在 Boss 已登录**。没 Boss 会话 cookie，扩展访问不到聊天。
- **`intake_candidate_id` URL 参数触发 orchestrator**：从 /intake「开始沟通」按钮跳转是这样的；直接开 Boss 不会触发。

## 如何看出问题

- 后端终端：看 `INFO` 日志有无 `POST /api/intake/collect-chat 200`
- 扩展：Boss 页右上角 `【采集】` toast 会说「正在分析聊天记录...」「简历已上传」「下一步: send_hard」
- 审计：`data/recruitment.db` 表 `audit_events` 所有 `f_stage LIKE 'f4_%'` 行
